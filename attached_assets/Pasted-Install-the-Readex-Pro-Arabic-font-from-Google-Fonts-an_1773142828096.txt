Install the Readex Pro Arabic font from Google Fonts and apply it across the entire website.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 1 — src/app/layout.tsx

Replace the IBM_Plex_Sans_Arabic import with Readex_Pro:

FROM:
import { Sora, DM_Sans, IBM_Plex_Sans_Arabic } from 'next/font/google'

const ibmArabic = IBM_Plex_Sans_Arabic({
  subsets: ['arabic'],
  weight: ['400', '500', '600'],
  variable: '--font-arabic',
  display: 'swap',
})

TO:
import { Sora, DM_Sans, Readex_Pro } from 'next/font/google'

const readexPro = Readex_Pro({
  subsets: ['arabic', 'latin'],
  weight: ['300', '400', '500', '600', '700'],
  variable: '--font-arabic',
  display: 'swap',
})

Then in the body className, replace ibmArabic.variable with readexPro.variable:
FROM:
<body className={`${sora.variable} ${dmSans.variable} ${ibmArabic.variable} font-sans antialiased`}>
TO:
<body className={`${sora.variable} ${dmSans.variable} ${readexPro.variable} font-sans antialiased`}>

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 2 — src/app/globals.css

Add Readex Pro as the arabic font utility class and update the CSS variable:

Add this inside the @layer utilities or at the bottom of the file:

.font-arabic {
  font-family: var(--font-arabic), 'Readex Pro', sans-serif;
}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 3 — tailwind.config (if exists, e.g. tailwind.config.ts or tailwind.config.js)

Make sure the font family is registered. Find the extend.fontFamily section and add:

arabic: ['var(--font-arabic)', 'Readex Pro', 'sans-serif'],

If the tailwind config does not have a fontFamily section yet, add:
theme: {
  extend: {
    fontFamily: {
      arabic: ['var(--font-arabic)', 'Readex Pro', 'sans-serif'],
    }
  }
}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
IMPORTANT NOTES:
- Do NOT change any other font (Sora and DM_Sans stay as-is)
- The CSS variable name stays --font-arabic so nothing else needs updating
- The font-arabic class is already used across all components — no need to touch them
- Readex Pro supports both Arabic and Latin characters perfectly
- After the change, restart the dev server so the new font loads correctly
